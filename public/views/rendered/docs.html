Instructions for using the built-in REST API

The API is simple to use, handles GET, POST, PUT, and DELETE requests, and communicates to a MongoDB database hosted by mLab. By using the API, users can query and modify a collection of sports teams. For now, users must use an application or browser extension to send certain HTTP requests to the API. I recommend Google Chrome's Postman extension. However, I'm working on a GUI interface to handle those requests. When it is completed, I will add it to the application.

Testing

By setting NODE_ENV='test' and starting the server, the application will attempt to connect to a local instance of MongoDB. This is used for running the Mocha tests contained in the test/test.js file which test all operations of the API. By starting the server with NODE_ENV='global', or without the NODE_ENV variable set, the application will connect to the MongoDB database hosted by mLab for this application. This functionality is accomplished by the config/index.js file.

GET [url]/teams

Returns an array of all objects in the collection

API Call: [url]/teams
Response: [{ "_id":012345,"name":"Yankees","City":"New York","League":"MLB" }]

GET [url]/teams/:team_id

Returns an object representing a unique team

API Call: [url]/teams/:team_id
Response: { "_id":012345,"name":"Yankees","City":"New York","League":"MLB" }

POST [url]/teams

Adds a team to the database

API Call: [url]/teams
Response: { "message":"Team created!",{ "_id":012345,"name":"Yankees","City":"New York","League":"MLB" }}

PUT [url]/teams/:team_id

Updates the information of an existing team

API Call: [url]/teams/:team_id
Response: { "message":"Team updated!",{ "_id":012345,"name":"Yankees","City":"New York","League":"MLB" }}

DELETE [url]/teams/:team_id

Deletes an existing team from the database

API Call: [url]/teams/:team_id
Response: { "message":"Successfully deleted" }
